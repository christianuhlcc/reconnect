# Realtime contract

All realtime data rides LiveKit. Two transport mechanisms, do not mix them up:

- **Data channels** carry frequent, transient events (position) and room-wide events (chat).
- **Participant metadata** carries each user's stable avatar identity (name/sprite/colour).

Encode/decode lives in `/lib/realtime.ts`. Every feature that touches the wire MUST use those helpers, never hand-rolled JSON.

## Message types

```ts
// topic "pos" â€” LOSSY data channel, sent at most every 100ms WHILE MOVING only
export type PosMsg = {
  t: "pos";
  x: number;        // Phaser world pixels (same coordinate space as the scene)
  y: number;
  dir: "up" | "down" | "left" | "right";
  moving: boolean;  // false on the final message when the avatar stops
};

// topic "chat" â€” RELIABLE data channel
export type ChatMsg = {
  t: "chat";
  id: string;       // uuid, generated by sender
  name: string;     // sender display name at send time
  body: string;     // trimmed, max 500 chars
  ts: number;       // Date.now()
};

export type WireMsg = PosMsg | ChatMsg;
```

```ts
// Participant METADATA (JSON-stringified) â€” not a data-channel message.
// Set at token-mint time (simplest) or via setMetadata at runtime.
export type AvatarMeta = {
  name: string;
  sprite: string;   // spritesheet key, e.g. "char_a"
  color: string;    // hex string, e.g. "#7cc4ff"
};
```

## Rules

- **Discriminator:** branch on `t`. Unknown `t` â†’ ignore silently (forward-compat).
- **Encoding:** payloads are `Uint8Array`. Use `TextEncoder`/`TextDecoder` over `JSON`. `/lib/realtime.ts` exports `encode(msg): Uint8Array` and `decode(bytes): WireMsg | null` (returns null on parse failure â€” never throw into the receive loop).
- **Topics:** always set the LiveKit `topic` (`"pos"` or `"chat"`) so receivers can route without parsing first.
- **Reliability:** `pos` â†’ lossy (`reliable: false`). `chat` â†’ reliable (`reliable: true`).
- **Position throttle:** broadcast `pos` at most ~10 Hz (â‰Ą100ms apart) and only while the avatar is moving; send one final `moving: false` message when it stops. Never send on a fixed timer when idle.
- **Validation:** clamp `body` to 500 chars on send; drop empty bodies. Treat all received fields as untrusted.

## Audio attenuation model (used by the proximity-audio skill)

```ts
export const FULL_RADIUS = 120;    // px: within this, gain = 1.0
export const CUTOFF_RADIUS = 360;  // px: beyond this, gain = 0 (silence)

export function gainForDistance(dist: number): number {
  const g = (CUTOFF_RADIUS - dist) / (CUTOFF_RADIUS - FULL_RADIUS);
  return Math.max(0, Math.min(1, g));
}
```

Keep these constants in `/lib/realtime.ts` so the office tile size can be tuned in one place.
